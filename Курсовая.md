# 1. ВВЕДЕНИЕ

<div align="justify">

Барометрический высотомер является важнейшим сенсорным элементом системы автоматического управления беспилотным летательным аппаратом. Он обеспечивает обратную связь по вертикальному каналу, позволяя выполнять автоматический взлёт, посадку, удержание высоты и возврат на базу (RTH).

В отличие от лидара или ультразвукового дальномера, барометр не зависит от отражательной способности поверхности и может работать на любой высоте в пределах тропосферы. В то же время он превосходит GPS по точности измерения относительной высоты.

В учебном полётном контроллере используется датчик давления MS5611 компании TE Connectivity. Благодаря высокой точности, низкому энергопотреблению и встроенной температурной компенсации данный датчик широко применяется в профессиональных автопилотах.

Цель работы — анализ архитектуры, протокола взаимодействия и программной реализации драйвера MS5611 в составе полётного контроллера.

</div>

# 2. ТЕОРЕТИЧЕСКАЯ ЧАСТЬ

## 2.1 Роль измерения высоты в БПЛА

<div align="justify">

Высотомер участвует в стабилизации полёта, автоматической посадке, удержании позиции и навигации.

Без точного канала высоты невозможно реализовать устойчивое зависание квадрокоптера.

</div>

---

## 2.2 Принцип работы пьезорезистивного барометра

<div align="justify">

Пьезорезистивный эффект заключается в изменении сопротивления полупроводника при механической деформации.

В датчике MS5611 используются четыре тензорезистора, объединённые в мост Уитстона.

При изменении давления мембрана деформируется, а встроенный 24-битный АЦП преобразует изменение напряжения в цифровой код.

</div>

---

## 2.3 Обзор датчика MS5611

| Параметр | Значение |
|---|---|
| Диапазон давления | 10–1200 мбар |
| Разрешение | 0,012 мбар |
| Время преобразования | 8,22 мс |
| Интерфейс | I²C / SPI |
| Напряжение питания | 1,8–3,6 В |

Особенность MS5611 – наличие энергонезависимой памяти PROM, в которую производитель записывает индивидуальные калибровочные коэффициенты. Благодаря этому возможно выполнение компенсации второго порядка без внешней калибровки, что критически важно для массового производства БПЛА.

## 2.4 Трёхцикловая схема опроса

<div align="justify">

Главный цикл управления работает с частотой 250 Гц, то есть период одного цикла составляет 4 мс.

Однако преобразование MS5611 при максимальном разрешении требует 8,22 мс.

Поэтому драйвер разбивает опрос датчика на три последовательных этапа:

</div>

- запуск измерения давления;
- чтение давления и запуск измерения температуры;
- чтение температуры и вычисление высоты.

<div align="center">

$$
f_{baro} = \frac{250}{3} \approx 83.3 \text{ Гц}
\tag{1}
$$

</div>

---

## 2.5 Температурная компенсация

<div align="justify">

Для компенсации температурной зависимости используются коэффициенты \(C1...C6\), записанные в PROM датчика.

</div>

<div align="center">

$$
dT = raw\_temperature - C5 \cdot 256
\tag{2}
$$

</div>

<div align="center">

$$
T = 20 + \frac{dT}{838}
\tag{3}
$$

</div>

<div align="center">

$$
OFF = C2 \cdot 65536 + \frac{C4 \cdot dT}{128}
\tag{4}
$$

</div>

<div align="center">

$$
SENS = C1 \cdot 32768 + \frac{C3 \cdot dT}{256}
\tag{5}
$$

</div>

<div align="center">

$$
P = \frac{\left(raw\_pressure \cdot SENS / 2097152\right)-OFF}{32768}
\tag{6}
$$

</div>

---

## 2.6 Методы расчёта высоты

<div align="center">

$$
h \approx (P_0 - P)\cdot0.0842
\tag{7}
$$

</div>

<div align="justify">

где:

- \(P_0\) — давление при старте;
- \(P\) — текущее давление.

</div>

<div align="center">

$$
h = 44330 \cdot \left[1-\left(\frac{P}{P_0}\right)^{0.1903}\right]
\tag{8}
$$

</div>

## 3.2 Функция `readBarometer()`

```cpp
void readBarometer() {

    barometer_counter++;

    if (barometer_counter > 3)
        barometer_counter = 1;

    if (barometer_counter == 1) {

        HWire.beginTransmission(0x77);
        HWire.write(0x48);
        HWire.endTransmission();
    }

    if (barometer_counter == 2) {

        HWire.beginTransmission(0x77);
        HWire.write(0x00);
        HWire.endTransmission();

        HWire.requestFrom(0x77, 3);

        raw_pressure =
            ((uint32_t)HWire.read() << 16) |
            ((uint32_t)HWire.read() << 8)  |
            ((uint32_t)HWire.read());

        HWire.beginTransmission(0x77);
        HWire.write(0x58);
        HWire.endTransmission();
    }

    if (barometer_counter == 3) {

        HWire.beginTransmission(0x77);
        HWire.write(0x00);
        HWire.endTransmission();

        HWire.requestFrom(0x77, 3);

        raw_temperature =
            ((uint32_t)HWire.read() << 16) |
            ((uint32_t)HWire.read() << 8)  |
            ((uint32_t)HWire.read());

        actual_altitude =
            (actual_pressure_start - actual_pressure) * 0.0842;
    }
}
```

---

## 3.3 PROM и хранение коэффициентов

<div align="justify">

Коэффициенты \(C1...C6\) считываются из энергонезависимой памяти PROM через интерфейс I²C.

Без калибровочных коэффициентов ошибка измерения давления может достигать десятков миллибар.

</div>

---

## 3.4 Ограничения текущей реализации

<div align="justify">

В текущем варианте программы отсутствует цифровая фильтрация высоты.

Из-за вибраций и шумов возникают колебания сигнала, что ухудшает стабильность зависания.

</div>

# 6. ПРЕДЛОЖЕНИЕ ПО УЛУЧШЕНИЮ

<div align="justify">

Для уменьшения шумов предлагается использовать экспоненциальный IIR-фильтр.

</div>

<div align="center">

$$
y[n] = \alpha \cdot x[n] +
(1-\alpha)\cdot y[n-1]
\tag{14}
$$

</div>

где:

- \(x[n]\) — текущее значение высоты;
- \(y[n]\) — фильтрованное значение;
- \(\alpha\) — коэффициент фильтрации.

```cpp
float filtered_altitude = 0.0;
const float ALPHA = 0.3;

if (filtered_altitude == 0.0)
    filtered_altitude = actual_altitude;
else
    filtered_altitude =
        ALPHA * actual_altitude +
        (1 - ALPHA) * filtered_altitude;
```
